# Benchmark Artifacts

This directory is the checked-in publication root for reproducible HIRN-Bench output.

Use it for publishable manual or nightly runs, not for ad hoc local smoke tests. PR smoke and developer sanity checks should continue to use the fast pseudo-embedding path unless you are intentionally publishing evidence.

## Recommended Filenames

- `cognitive.md` and `cognitive.json`
- `locomo.md` and `locomo.json`
- `dmr.md` and `dmr.json`
- `longmemeval.md` and `longmemeval.json`
- `load.md` and `load.json`
- `advanced.md` and `advanced.json`
- `advanced-history.json`
- `advanced-baseline.json`

## Required Metadata For Publishable Runs

- generated-at timestamp / run date
- git SHA or release tag
- Cargo.lock blake3 (or equivalent dependency lock fingerprint)
- dataset/source name
- embedding model label
- environment label
- reproducibility drift summary when `--runs > 1`

## Review Rule

Target tables in the docs are goals. Competitor-comparison or benchmark-leadership claims should only be made when the corresponding artifact generated by the documented commands is checked in here and reviewable in git.

When publishing paired Markdown/JSON/CSV artifacts for the same run, prefer one benchmark execution plus companion output flags such as `--json-output` or `--markdown-output`. That keeps `run_id`, generated-at time, and measured results identical across the emitted files.
